# 0020. `.github`リポジトリ自身も共通標準の適用対象とする

## Metadata（メタデータ）

| 項目 | 内容 |
| --- | --- |
| Status | Accepted |
| Date | 2026-07-14 |

---

## Context（背景）

共通標準を定義・提供する`.github`リポジトリ自身が、その共通標準に従うべきかどうかが論点になった。標準を定義する側であることを理由に、`.github`リポジトリを例外扱いする考え方もありえた。

---

## Decision（決定）

`.github`リポジトリ自身も、このプロジェクトが定義する共通標準（Philosophy・Architecture・Convention・Specification）の適用対象とする。

---

## Rationale（理由）

標準を定義するリポジトリを例外扱いすると、「共通標準は他のリポジトリには適用するが、定義元には適用しない」という二重基準が生まれる。これは、共通標準そのものの信頼性・実効性を損なう。`.github`リポジトリ自身が標準に従うことで、標準が実際に運用可能であることを示す実例にもなる。

---

## Alternatives Considered（却下した案）

| 選択肢 | 採用しなかった理由 |
| --- | --- |
| `.github`リポジトリを標準の適用対象外とする（定義元としての特例を認める） | 標準を定義する側とされる側で異なる基準を持つことになり、共通標準全体の一貫性・信頼性が損なわれる |

---

## Consequences（影響・注意点）

- `.github`リポジトリであること自体を理由に、適用対象となる共通標準から除外しない。このリポジトリに適用されるConvention・Specificationについては、他のリポジトリと同じ基準で従う
- `.github`リポジトリ固有の事情がある場合は、他のリポジトリと同様にRepository Overrideとして明示する（ADR 0009参照）
