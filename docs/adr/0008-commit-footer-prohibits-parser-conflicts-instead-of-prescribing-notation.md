# 0008. Commit Footerは表記を指定せず解析競合回避の制約のみ定義する

## Metadata（メタデータ）

| 項目 | 内容 |
| --- | --- |
| Status | Accepted |
| Date | 2026-07-14 |

---

## Context（背景）

Commit MessageのFooterで`BREAKING CHANGE:`の説明本文に補助的な項目（例：「移行方法」）を書く際の表記方法を検討する過程で、当初「区切りに半角コロンではなく全角コロン（：）を使う」という具体的な表記案が検討された。理由は、説明文中の半角コロンが、Git・GitHub・CI等の解析ツールによって`key: value`形式のFooterトークンとして誤認識される可能性があったためである。

しかしこの案は、全角・半角の区別を持たない英語表記に適用できないことが判明した。問題を掘り下げた結果、本質的な課題は「全角コロンを使うかどうか」ではなく、「Footerの解析器と競合する`key: value`的な構造を、説明本文中に新たに作らないこと」であると整理された。

---

## Decision（決定）

Commit Footer内の説明表現について、特定の記法を強制しない。解析ツールがFooterの`key: value`構造として誤認識しうる独自の構造を、説明本文内に新たに作成してはならない、という制約のみを定義する。

---

## Rationale（理由）

書き方（使用する記号・記法）を具体的に指定すると、言語（日本語・英語等）ごとに異なる対応が必要になり、規則が複雑化し、特定の言語に依存したものになる。

本質的な課題は、特定の文字や記法の使用可否ではなく、解析ツールとの互換性（Footerパーサーが誤認識するかどうか）である。制約を「してはならないこと」に絞ることで、言語に依存せず、書き手が表現方法（見出し・箇条書き・改行等）を自由に選べる状態を保てる。

---

## Alternatives Considered（却下した案）

| 選択肢 | 採用しなかった理由 |
| --- | --- |
| 全角コロン（：）を用いて区切る | 半角コロンによる`key: value`誤認識を防ぐ具体策として検討されたが、全角・半角の区別がない英語表記には適用できず、言語依存の解決策になってしまう |
| 見出しや箇条書きなど、特定の表現方法を規約として指定する | 特定の表現方法を固定すると、将来別の合理的な表現を取りたい場合に規約自体の変更が必要になる。禁止事項だけを定義する方が、表現の自由度を保ちながら同じ目的を達成できる |

---

## Consequences（影響・注意点）

- Commit Footerの説明文では、新たな`key: value`的な区切り構造を作らない限り、見出し・箇条書き・改行など任意の表現方法を選べる
- 具体的な禁止事項の詳細（Footerのキーバリュー形式、曖昧なキーの禁止等）は`docs/conventions/commit.md`が定義する
